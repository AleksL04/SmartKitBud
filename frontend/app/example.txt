========================================
Tailwind CSS Component Snippets
========================================

A collection of common UI component class strings for Tailwind CSS.

---
Component: 1. Centered Page Layout
---

Purpose: A standard layout for page content, centered with a maximum width.

Elements:
  <body>
    Classes: bg-gray-100 text-gray-800
    Notes:   Sets a light gray background and dark gray text for the whole page.

  Main Container <div>
    Classes: container mx-auto p-4 sm:p-6 lg:p-8
    Notes:   Creates a responsive, centered container with padding that increases on larger screens.

  Content Box <div>
    Classes: max-w-4xl mx-auto bg-white p-8 rounded-lg shadow-md
    Notes:   A white card with a max-width, centered, with padding, rounded corners, and a shadow.

  Main Heading <h1>
    Classes: text-3xl font-bold mb-4
    Notes:   Large, bold text with a bottom margin.

  Paragraph <p>
    Classes: text-lg
    Notes:   Slightly larger text for readability.

---
Component: 2. Responsive Navigation Bar
---

Purpose: A navigation bar that is horizontal on desktop and stacks on mobile.

Elements:
  <nav> (Navbar Container)
    Classes: bg-gray-800 text-white p-4
    Notes:   A dark background with white text and padding.

  <div> (Inner Container)
    Classes: container mx-auto flex justify-between items-center
    Notes:   Centers content and uses flexbox to space out the logo and menu items.

  <a> (Logo)
    Classes: text-xl font-bold
    Notes:   Larger, bold text for the brand.

  <button> (Mobile Menu Button)
    Classes: md:hidden
    Notes:   Visible on small screens, hidden on medium screens and up.

  <div> (Desktop Menu Container)
    Classes: hidden md:flex space-x-4
    Notes:   Hidden on small screens, becomes a flex container on medium screens and up. `space-x-4` adds horizontal space between links.

  <a> (Menu Links)
    Classes: hover:bg-gray-700 px-3 py-2 rounded
    Notes:   Adds padding, rounded corners, and a background color change on hover.

---
Component: 3. Responsive Card Grid
---

Purpose: A flexible grid of cards that adjusts the number of columns based on screen size.

Elements:
  <div> (Grid Container)
    Classes: grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8
    Notes:   Creates a grid. 1 column on mobile, 2 on medium screens, 3 on large. `gap-8` adds space between items.

  <div> (Card)
    Classes: bg-white rounded-lg shadow-lg overflow-hidden
    Notes:   A white card with rounded corners, a larger shadow, and `overflow-hidden` to round the image corners.

  <img> (Card Image)
    Classes: w-full h-48 object-cover
    Notes:   Full width, fixed height, and covers the area without distortion.

  <div> (Card Content Wrapper)
    Classes: p-6
    Notes:   Adds padding inside the card, below the image.

  <h3> (Card Title)
    Classes: text-xl font-semibold mb-2
    Notes:   Styled text for the card's title.

  <p> (Card Description)
    Classes: text-gray-600
    Notes:   Slightly lighter text for the description.

---
Component: 4. Simple Form Input
---

Purpose: A styled text input field with a label, including focus states.

Elements:
  <label> (Input Label)
    Classes: block text-gray-700 text-sm font-bold mb-2
    Notes:   A standard label style placed above the input.

  <input type="text"> (Text Input)
    Classes: shadow appearance-none border rounded w-full py-2 px-3 text-gray-700 leading-tight focus:outline-none focus:shadow-outline focus:border-blue-500
    Notes:   A clean input style. The `focus:` classes are crucial for accessibility, providing visual feedback when the user interacts with the field.
